## Core
- Added demand power data collection for channels with trend collection enabled
- Added diagnostic trend data export to Parquet file when generating weekly reports
- Added automatic deletion of old report Parquet files (maximum retention: 52 weeks)
- Added 15-minute interval energy trend collection thread
- Added leakage current to trend collection parameters
- Added DO alarm state calculation and storage during 5-minute diagnostic state saves
- Added training data Parquet file export, running as a separate process
- When storing diagnostic data, if AssetDriveType = VFD, THD/TDD trend data is retrieved through the getRealtimeData API and stored in Redis (THD_V, THD_I, TDD_I).
- When storing meter data, the average value is calculated, and the THD/TDD trend data is stored in Redis (THD_V, THD_I, TDD_I).
