## Core
- トレンド収集が有効なチャンネルのデマンド電力データ収集機能を追加
- 週次レポートファイル生成時、診断トレンドデータをParquetファイルに保存する処理を追加
- 古いレポートParquetファイルの自動削除機能を追加（最大保存期間：52週）
- 15分周期のエネルギートレンド収集スレッドを追加
- トレンド収集パラメータに漏洩電流を追加
- 5分周期の診断状態保存時、DOアラーム状態の計算・保存処理を追加
- 学習データParquetファイルの保存処理を追加（別プロセスとして動作）
- 診断データ保存時、AssetDriveType = VFD の場合、getRealtimeData API を通じて THD/TDD トレンドデータ を取得し、Redis（THD_V, THD_I, TDD_I）に保存する。
- メーターデータ保存時、平均値を計算し、THD/TDD トレンドデータを Redis（THD_V, THD_I, TDD_I）に保存する。

## WebServer
- 設定：デマンド電力トレンド収集オプションを追加、有効時にダウンサンプリングタスクを追加
- 設定：初回インストール時に実行されるダウンサンプリングタスクにエネルギートレンドタスクを追加
- 設定：Modbus TCP/Serialログ有効化オプションを追加
- 設定：General – Modbus Serialのモジュール情報入力欄を追加
- トレンド：デマンド電力トレンド収集が有効な場合、デマンド電力タブを追加
- トレンド：15分電力量トレンドタブを追加 / 計測タブに主要パラメータを追加
- サービス：Parquetファイルダウンロード機能を追加、学習データParquetファイルダウンロード機能を追加
- サービス：トレンドデータCSVファイルダウンロード機能を追加
- ダッシュボード：シリアルModbusモジュール通信状態の表示を追加
- メンテナンス：全ビルドバージョンおよびリリースノート表示機能を追加
- トレンド: AssetDriveType に応じて ツリーテーブルの分岐表示を実装 / ツリーテーブルのリファクタリング。
- ダッシュボード / 計測: AssetDriveType に応じて THD の分岐表示を実装。
